# Twitter/X — Skills Trap
*Source: "The Claude Code Skills Trap (Most People Fall For This)" — YouTube*

---

## Thread 1: Main Thread (Value)

**Tweet 1 (Hook):**
You're making your Claude Code agent less smart every time you install a new skill.

Not a little less smart. Like, noticeably worse.

Here's why (and the 5-min fix):

**Tweet 2:**
Trap 1: Hoarding.

Claude has a 15,000 character budget for its skills list.

Every skill's name + description gets crammed in there.

200 skills = a noisy menu where Claude picks the wrong one half the time.

Less noise = better output. Every time.

**Tweet 3:**
The fix: curate, don't collect.

20-30 well-built skills specific to your workflows.

If you can't explain when Claude should use one skill vs another, delete one of them.

**Tweet 4:**
Trap 2: Copy-pasting marketplace skills without adding your context.

You install a copywriting skill. Run it. Output is... fine.

Generic. Competent. Forgettable.

Because it knows nothing about your brand, your audience, or how you actually talk.

**Tweet 5:**
The fix: 15 minutes.

Install locally (--local flag).
Add a reference file with your brand voice.
Drop in 3-5 examples of what good output looks like.
Tell the SKILL.md to read them first.

That's it. Generic skill → your skill.

**Tweet 6:**
Trap 3: The Monolith.

Brand voice, examples, scoring sheets, process steps — all in one massive SKILL.md.

Feels organised. But Claude loads every token of it every single run.

When the output's wrong, you're staring at one giant wall of text with no idea what to fix.

**Tweet 7:**
The fix: separate process from knowledge.

SKILL.md = what Claude does, step by step.
references/ = brand context, examples, rules.

Claude loads only what it needs. You know exactly where to fix things.

**Tweet 8:**
TL;DR:

- Curate 20-30 skills (not 200)
- Add your brand context to every skill you install
- Separate process from knowledge (SKILL.md vs references/)

The best skill setup isn't the biggest. It's the one that's built for YOUR business.
