# Threads — Skills Trap
*Source: "The Claude Code Skills Trap (Most People Fall For This)" — YouTube*

---

## Post 1: Take

Something I keep noticing in Claude Code setups:

People install 50-200 skills thinking more = more capable.

But Claude has a 15,000 character budget for the skills list. Every name and description competes for space. More skills = noisier menu = Claude picks the wrong one.

You're literally making your agent dumber by installing more.

20-30 curated skills. That's the sweet spot. What's yours at?

---

## Post 2: Mini-Thread

**Post 1:** Three traps I see in almost every Claude Code skill setup right now.

**Post 2:** Trap 1 — Hoarding. 200 installed skills looks impressive but Claude can't navigate a noisy menu. Curate 20-30 that are specific to your workflows. Delete the rest.

**Post 3:** Trap 2 — Copy-pasting marketplace skills without adding your business context. It's like hiring a consultant who's never been briefed. 15 minutes of customisation changes everything.

**Post 4:** Trap 3 — Cramming process + context + examples into one SKILL.md file. Separate them. SKILL.md for process, references/ for knowledge. Claude loads only what it needs.

**Post 5:** Bottom line — the best skill setup isn't the biggest. It's the one built for YOUR business.

---

## Post 3: Observation

280,000+ skills on SkillsMP.

Most are auto-generated by an LLM. One file, no references, no folder structure.

And people install dozens of these and wonder why their agent feels generic.

The skill.md is just the process. The reference files you add around it are where the actual value is. That's the bit everyone skips.
