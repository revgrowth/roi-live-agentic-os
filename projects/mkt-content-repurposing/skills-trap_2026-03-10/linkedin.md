## Post 1: Text Post (Value)

I reviewed dozens of Claude Code skill setups last month.

Almost everyone is making the same three mistakes. And they're blaming the skills.

But the skills work. It's the setup that's broken.

Here's what I keep seeing:

1. **Hoarding skills.** People install 50, 100, even 200 skills. Looks impressive. But Claude has a 15,000 character budget for its skills list. The more you install, the noisier the menu. Claude picks worse. Your output gets inconsistent. You're literally making your agent less smart by installing more.

2. **Copy-pasting without context.** You grab a copywriting skill from a marketplace. Run it. The output is... fine. Generic. Competent. Like ChatGPT 3.5. Because that skill has zero context about your brand, your audience, or how you actually talk. It's an unbriefed consultant giving textbook answers.

3. **Cramming everything into one file.** Brand voice, examples, scoring sheets, process steps — all in one massive SKILL.md. It feels tidy. But Claude loads the entire thing every time, wastes tokens on stuff it doesn't need, and when the output is wrong? Good luck figuring out what to fix.

The fix for each one takes about 15 minutes.

Curate 20-30 skills. Add your brand context to the reference files. And separate your process from your knowledge.

The best skill setup isn't the biggest. It's the one where every skill is personalised to your business.

What does your current skill setup look like — curated or collected?

## Post 2: Carousel Outline (Value)

**Slide 1:** "You're making your Claude Code skills LESS smart"
*Swipe to see the 3 traps (and the 5-min fix for each)*

**Slide 2:** "Trap 1: The Hoarding Trap"
Claude has a 15,000 character budget for skills. 200 installed skills = noisy menu = worse picks = inconsistent output.

**Slide 3:** "The Fix"
Audit what you've got. Build a curated list of 20-30 skills specific to YOUR workflows. If you can't explain when Claude should use one vs another, delete one.

**Slide 4:** "Trap 2: The Copy-Paste Trap"
Marketplace skills are built for the general public. No brand voice. No audience data. No examples. Like hiring a consultant who's never been briefed.

**Slide 5:** "The Fix"
Install locally (--local flag). Add a reference file with your brand context. Drop in 3-5 examples. Update the SKILL.md to point to them. 15 minutes.

**Slide 6:** "Trap 3: The Monolith Trap"
Everything in one SKILL.md file. Process + context + examples + rules = one giant wall of text. Claude loads it all every run. You can't debug it. You can't iterate.

**Slide 7:** "The Fix"
SKILL.md = process only (what Claude does, step by step). References/ = knowledge (brand voice, examples, rules). Claude loads only what it needs, when it needs it.

**Slide 8:** "The best skill setup isn't the biggest."
It's the one where every skill is personalised to your business and built so you can improve it.
