<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Cron runtime -> Claude CLI | Client cron runs must carry a stricter launch contract than root cron runs | prompt, model, permission mode, allowed directory, cwd |
| Client workspace -> root workspace | A client cron run must not silently inherit broad root access just because it is launched from the same repo | filesystem reads, listings, writes |
| Scheduled/manual trigger -> shared execution path | Manual client runs must remain separate tasks but still use the same containment rules | trigger type, task row, cron run row |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Privilege escape | client cron launch args | mitigate | Stop using `--dangerously-skip-permissions` for client cron runs and build an explicit client-only launch spec |
| T-03-02 | Cross-workspace read exposure | client cron session | mitigate | Pass the selected client workspace as the allowed directory boundary in the Claude launch args |
| T-03-03 | Regression to root cron behavior | root cron path | mitigate | Keep root cron launch behavior unchanged and cover it with launch-spec tests |
</threat_model>
