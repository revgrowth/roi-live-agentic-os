<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Client workspace -> rest of repo | Anything outside the selected client folder must be treated as out of bounds | file writes, file edits, generated outputs |
| Post-run detector -> failure finalizer | If an escaped mutation is detected, the failure must go through the shared runtime truth path | offending paths, result, error detail |
| Selected client workspace -> allowed writes | The detector must not reject legitimate writes inside the chosen client folder | client-local projects, context, cron, brand files |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Cross-client contamination | post-run mutation check | mitigate | Detect any mutation outside the selected client subtree, not just root output folders |
| T-03-05 | False positive containment failure | allowed client writes | mitigate | Treat the full selected client folder as the allowed write boundary |
| T-03-06 | Silent policy break | failure surfacing | mitigate | Send escaped-write failures through the existing unified cron failure finalizer |
</threat_model>
