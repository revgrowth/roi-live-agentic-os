<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Scheduler or queue watcher -> process manager | Queued task events can arrive more than once for the same task before execution starts | task id, queued status, cron job metadata |
| Process manager -> Claude child process | One task claim should create one child process and one prompt execution | prompt text, cwd, permission mode |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | queue watcher and process manager start path | mitigate | Claim queued execution before async work and reject duplicate start attempts for the same task id |
| T-01-02 | Tampering | task chat and run history | mitigate | Ensure only one Claude start occurs per queued scheduled task so one trigger cannot append multiple reply sequences |
</threat_model>
