## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Process manager -> cron runtime helper | Task completion data should cross one shared boundary instead of being rewritten by two owners | task id, result, cost, duration, client id |
| Cron runtime retry loop -> Claude child process | The same scheduled prompt must not be replayed more than once automatically | prompt text, trigger type, retry count |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Tampering | cron run history and status files | mitigate | Remove direct `cron_runs` and status-file writes from process-manager and route completion through `completeCronRunForTask(...)` |
| T-01-04 | Denial of Service | scheduled retry loop | mitigate | Cap scheduled executions to two total attempts and keep retries inside the same task/run record |
</threat_model>
