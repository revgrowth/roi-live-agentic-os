<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Windows launcher -> detached daemon | Start command should not create a lingering visible cron terminal or change daemon behavior | daemon PID, log path, runtime leadership |
| Cron runtime -> Claude CLI | Scheduled cron execution must use a hidden launch path that still receives the right environment | Claude command path, prompt args, Git Bash path, timeout |
| Cron runtime -> fallback wrapper | Script-based overrides may still need a helper, but only as a narrow fallback | File path, env overrides, stdout/stderr temp files |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | Windows scheduled launch path | mitigate | Prefer the proven direct hidden Claude launch path and avoid the regression wrapper for normal `claude.exe` execution |
| T-02-02 | Availability | detached cron environment | mitigate | Discover and pass an existing Git Bash path automatically so background cron runs do not fail only because shell state was lost |
| T-02-03 | UX regression | Windows daemon start | mitigate | Make start behavior explicitly fire-and-forget and remove misleading "terminal stays attached" wording |
</threat_model>
