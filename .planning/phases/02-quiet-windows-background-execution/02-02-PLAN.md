<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Cron outcome logic -> task and cron run state | Every result must update runtime truth before user-facing surfacing | task status, cron result, duration, exit code, error text |
| Cron runtime -> Windows notification helper | Desktop notification calls must reflect the exact cron outcome without inventing a second truth source | event type, job name, client label, failure detail, duration |
| Job notify setting -> notification dispatch | Per-job notify policy must control whether the helper is called | `on_finish`, `on_failure`, `silent` |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Repudiation | cron failure surfaces | mitigate | Route all execution-blocking failures through one finalization helper that updates status, logs, and notifications together |
| T-02-05 | Availability | background cron monitoring | mitigate | Send desktop notifications for blocker failures so hidden runs do not fail silently |
| T-02-06 | Tampering | per-job notification policy | mitigate | Enforce `on_finish`, `on_failure`, and `silent` in one helper instead of ad-hoc call sites |
</threat_model>
